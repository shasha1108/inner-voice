## 确认选项

> **请选择操作：**
> 1. ✅ 确认（触发持久化：写资产库 → 同步 github → 写飞书）
> 2. ✏️ 修改文案（告诉我修改方向）
> 3. 🎨 修改 Brief（告诉我修改方向）
> 4. 🚫 根因级否决（整体牵强 / 读者看不懂 / 结构性问题）
> 5. 🔄 全部重新生成

## 反馈处理

| 反馈类型 | 处理动作 |
|---------|---------|
| ✅ 直接确认 | 进入 STEP 8 最终持久化 |
| ✏️ 细节调整 | 局部修改 → 重新质检 → 再次展示确认点 B（不写任何文件） |
| 🚫 根因级否决 | 不入资产库 + 将根因写入 workflow-memory.md + 触发自检模型反思 |
| 🔄 全部重新生成 | 回到 STEP 2-A 或 STEP 1.5 |

**关键铁律：** 选项 1 之外的任何反馈，都不触发持久化。
