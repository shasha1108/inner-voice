# 隐喻候选输出模板

> 用于 STEP 3 人工确认点 A-1 输出。内部推演（身体经验→悖论提取→隐喻命名→视觉验证）不向用户展示。

## 输出格式

```
【候选隐喻列表】（至少3个，最多5个）
1. [隐喻名称「」] — 悖论：[一句话描述内在矛盾] — 视觉主体：[MASSIVE物理实体] — 对应读者体感：[一句话] — 美感来源：[为什么这个画面是美的] — 新颖度：高/中

【终选】
selected_metaphor = [入选的那一个]
入选理由：[悖论击中力 + 视觉美感 + 与已有隐喻的差异]
```

## 字段说明

| 字段 | 要求 |
|------|------|
| 隐喻名称 | 用「」标注，是读者已有的体感命名，非创造 |
| 悖论 | 两种对立状态同时为真，非物理形状描述 |
| 视觉主体 | 能占画面60%+的单一物理实体 |
| 读者体感 | 日常物理经验中本来就有的事 |
| 美感来源 | 必须能说清"为什么这个画面是美的"，说不清=不适合 |

## 否决后处理

1. 先输出诊断：`「我之前认为[X]有效的原因是[Y]，实际问题是[Z]」`
2. 不重做身体经验步骤
3. 回到悖论提取或隐喻命名步骤，增量修正
4. 只重新生成候选列表
