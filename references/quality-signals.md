
# 文案质量校验

> **使用方法：** STEP 6 加载。不测量"好不好"，只验证公式执行到位没有。所有检查项基于真实爆款逆推，不是主观品味。**仅检查文案，不涉及视觉。**

> ⚠️ **优先级硬约束**：G7（物理现实校验）优先级高于 G1-G6。**形式校验全部通过 ≠ 现实成立**——物理动作与心理意象矛盾时，文字形式再完美也是掩耳盗铃式伪共鸣。任何 G1-G6 通过但 G7 失败 = 仍打回。

---

## 校验一：硬性规则（9项，违反任一 → 打回）

| # | 检查项 | 验证方式 |
|---|--------|---------|
| **G1 禁用词** | grep 全文：`你应该\|\b学会(?!了)\|\b记住(?!了)\|请记得\|很多人都\|我们可以称之为\|心理学上称之为\|那个负责X的你\|这其实是\|本质上是一种\|不是[^也]*而是`。标题额外检查：不得使用 `«»` 书名号做标签装饰。命中 → 打回 |
| **G2 结尾非处方** | 取最后 3 句，检查含：`你应该\|可以尝试\|试试\|建议你\|希望你能`。命中 → 打回 |
| **G3 「」有引出** | 如全文用了「」命名：「」出现前 1-2 句须有铺垫（描述→命名，或前句主语自然过渡）。无铺垫凭空出现 → 打回。全文无「」→ 自动跳过 |
| **G4 具体锚点** | 全文至少含一类：直接引语（"想开点"）/ 具体数字（37次）/ 时间标记（凌晨3点）/ 身体动作（熄了火）/ 物理地点（工位）。零 → 打回 |
| **G5 视角不漂移** | 同段落内混用"我们"+"你"（诊断式）→ 打回。跨段落切换允许 |
| **G6 标题专项** | 三项：(1) 竖线标题（如使用）前后须能读成同一句连续的话，断裂 → 打回；(2) 标题含 ≥1 个天然搜索场景词（非硬塞）；(3) 标题中的核心词与 confirmed_metaphor 命名方向一致。违反 (2) → 警告；违反 (1) 或 (3) → 打回 |
| **G7 物理现实校验（最优先）** | 提取文案中的物理动作，放进现实社交场景中测试。如果该动作（如：面对面聊天时双手捂死耳朵5秒）在现实中显得极其荒谬、抓马或必然被对方打断，但文案却描写"无人察觉"，则判定为逻辑崩塌 → 打回重写。**形式校验全部通过 ≠ 现实成立** |
| **G8 隐喻路径二选一强制选择** | 隐喻必须在写作前预声明走"实写"或"虚写"路径之一，全文不可混用。**实写**：场景必须在私密空间（厕所、独处的车内、关门的房间），物理动作真实发生；**虚写**：表面动作必须极其微小、符合社交常规（深呼吸、短暂走神），"隐喻动作"只能在心里发生。混用 = 掩耳盗铃式伪共鸣 → 打回 |
| **G9 场景单一深挖** | 描写具体场景时，严禁报菜名式堆砌（同时出现手机响+亲戚催+老板找 = 缝合怪场景）。必须只选取 1 个极其微小、具体的物理切片（例：只是看着手机屏幕上对方正在输入的省略号闪烁），把 1 个细节写出厚度。多个具体元素在同一时点并发 = 场景崩塌 → 打回 |

### G1 正则说明

**禁用词模式解释**：
- `你应该` — 直接说教，无例外
- `\b学会(?!了)` — 匹配"学会X"（祈使/建议），排除"学会了"（过去描述）
- `记住(?!了)` — 匹配"记住X"（祈使），排除"记住了"（过去描述）
- `请记得` — 直接指令，无例外

**上下文判定（grep 命中后必须人工确认）**：
- `学会`/`记住` 在描述性语境中（主语为"我"，时态为过去/完成）→ 误报，手动放行
  - ❌ grep命中但放行：「我终于学会了放手」「我记住了那个瞬间」
  - ✅ grep命中且打回：「学会独处」「记住这三件事」「你要学会放下」

**「不是X而是Y」判断**：
- **命中**（禁止）:
  - `不是脆弱，而是敏感` → 两段式简单翻转，非递进
  - `不是你的错，而是你太在意` → `不是[^也]*而是` 命中

- **不命中**（允许）:
  - `不是难过，也不是崩溃，而是处在一种灰度状态` → 中间有"也"，是递进式否定链
  - `不是做太多了。是一直在表演不累` → 句号断开，两句独立陈述

## 校验二：结构验证

> 根据 STEP 4 确认的叙事角度，执行对应验证。
>
> **所有类型通用：** 隐喻用法一致性检查 → 方法定义和动词-宾语匹配规则见 `references/writing-techniques.md`「隐喻用法一致性检查」。违反 → 打回。

### Type A（感知细节）: 场景 → 隐喻 → 矛盾 → 重框

| 要素 | 验证 |
|------|------|
| 开场为感知场景或身体体验 | 前 100 字含空间/时间/身体的具象描写 |
| 矛盾句 ≥ 3 组 | 计数"X，却Y""X，但Y""明明X，偏偏Y"句式，每组两侧为完整句 |
| 结尾姿态为接纳/重框 | G2 已覆盖（无处方词）。额外检查：结尾是否在"给答案"——有结论但不说教 |

### Type B（反向洞察）: 挑战假设 → 反向逻辑 → 揭示我们漏掉了什么

| 要素 | 验证 |
|------|------|
| 开篇挑战了一个常见认知 | "我们总以为..." / "以前以为..." / 普遍认知被点名 |
| 反转逻辑完整 | 有"旧看法→为什么不对→新看法"的路径，非一句话翻转 |
| 「」非必须 | 反转本身完成了命名时不需要「」。如使用了「」→ G3 正常检查 |
| 结尾不重复反转 | 反转在中段完成，结尾不是再说一遍——结尾是"看到了什么新的" |

### Type C（悖论张力）: 悖论陈述 → 张力强化 → 接受，不解决

| 要素 | 验证 |
|------|------|
| 开篇含一个内在悖论 | "看似A，实则B"是开场内容，不是结尾总结 |
| 心理型矛盾句占比 | 统计矛盾句中"心理型"（暴露读者未意识到的内部状态）vs"修辞型"（A和非A对置）的数量。心理型 ≥ 1 组 |
| 结尾不化解悖论 | 不把悖论"解开"。让它成立——"就是这样，不解决" |

### Type D（功能归因）: 场景 → 隐喻 → 归因 → 矛盾 → 重框

| 要素 | 验证 |
|------|------|
| 归因段长度 | 1-2 句。超过 → 打回 |
| 归因语气 | 同行者分享（"也许..." "可能..."）。术语直译（"心理学上称之为..."）→ 打回 |
| 术语有体感翻译 | 如出现术语（杏仁核/HPA轴），须紧跟体感翻译（"大脑里的警报器"）。术语裸奔 → 打回 |
| 行动锚点来源 | 如含行动锚点，来源须为：(a) STEP 0 采集的用户亲测方法 (b) success-patterns.md 同类词条 (c) 用户对话中提供。自行发明 → 打回 |
| 行动锚点具体度 | 含时间/次数/具体动作（"20分钟"/"重复三次"/"调成灰度"）。泛化建议（多喝水/好好睡觉）→ 打回 |
| 行动锚点位置 | 仅在全文最后 1-3 行。提前出现 → 打回 |

## 校验三：写作指标

> 报数，不判死。目的：让写作模式可见，发现异常值。

| 指标 | 计算方式 | 作用 |
|------|---------|------|
| **锚点多样性** | 引语/数字/时间/身体/地点——各类型命中数 | 只有一种锚点类型 = 写作套路化 |
| **矛盾句密度** | 矛盾句组数 ÷ 全文句数 | 叙事型参考 0.08-0.18，钩子型参考 0.12-0.25。超出提示 |
| **结尾类型** | 分类：画面悬停 / 压缩定义 / 重框接纳 / 行动锚点 / 解释自己隐喻 / 给建议 | 后两种（解释隐喻、给建议）→ 警告 |
| **CTA 状态** | 有/无 + 类型（A/B选择/场景认领/开放邀请/连载关注/无）| 仅报告。不强制 |
| **语言均匀度** | 最短句字数 ÷ 最长句字数 | 比值 < 0.33（差距不到3倍）→ 提示"语言过于均匀，可能被抛光过" |
| **细节特异性** | 换掉情绪主题词（如"焦虑"→"孤独"），场景是否还能直接用 | 能 → 提示"场景为通用模板，缺乏独特性" |

## 5 个结构信号

> 逐一检查。触发 ≥ 2 个 → 内容有模板生产嫌疑 → 回到 STEP 5，优先解决信号指出的问题，而非修补单个指标。

### 信号 1 · 结论可预测性

> **读完标题，能猜到这篇的结论吗？**

| 判断 | 标准 |
|------|------|
| ✅ 通过 | 结论里有至少一处，在标题里完全猜不到——不是套路收尾，是真正想通了一个东西 |
| ❌ 触发 | 标题就是结论的压缩版——读完标题就知道结尾会说什么，中间只是填充 |

**检查方法：** 先读标题，猜一个结论。再读结尾。一样 → 触发。

---

### 信号 2 · 中间转折

> **正文中间有没有至少一处思考「拐弯」的地方？**

| 判断 | 标准 |
|------|------|
| ✅ 通过 | 有一处"但我后来发现...""其实不是这样的...""后来才想通..."——不是模板句式，是真的推进了一层 |
| ❌ 触发 | 每段严格对应一个结构功能（描述现象 → 揭示原因 → 给出新视角），没有多余的、意外的内容 |

**检查方法：** 逐段问"这一段如果删掉，前后文逻辑会断吗？"——如果每段都可以删而逻辑不断，说明段落之间没有思考推进，只是按模板排列。

---

### 信号 3 · 细节特异性

> **正文里有至少一个「只有这篇才会出现」的具体细节吗？**

| 判断 | 标准 |
|------|------|
| ✅ 通过 | 有一个时间/地点/动作/物件的组合过于具体，不像是为了凑场景编出来的 |
| ❌ 触发 | 场景全部是通用模板——换成另一个情绪主题也能直接用 |

**检查方法：** 把情绪主题词换掉（如"焦虑"换成"孤独"），场景还能直接用吗？能 → 触发。

---

### 信号 4 · 语言均匀度

> **全篇句子长度、节奏、精致度高度统一吗？**

| 判断 | 标准 |
|------|------|
| ✅ 通过 | 有粗糙的地方——某句话太短了、太长了、太口语了——但因为它是真实的，读起来不觉得突兀 |
| ❌ 触发 | 全篇像被抛光过——每句话差不多的长度、差不多的精致度、没有一句"出格"的 |

**检查方法：** 找全篇最短的一句和最长的句子。差距 < 3 倍字数 → 语言过于均匀 → 触发。

---

### 信号 5 · 结构要素挤压感

> **正文的每一个段落是否都在"完成一个结构任务"？**

这是校验二（结构验证）的副作用——agent 为了满足 Type 结构要素而生产出结构完美的文案，但每个要素之间没有思考推进，只有任务切换。

| 判断 | 标准 |
|------|------|
| ✅ 通过 | 段落之间的过渡是自然推进——"然后我开始想…""后来才意识到…"——下一个想法是从上一个想法里长出来的，不是从检查表里勾出来的 |
| ❌ 触发 | 至少两处段落过渡有"齿轮切换"感——这段在完成"反常识开场"、那段在完成"愤怒转折"、下一段在完成"重新框架"——结构要素的边界清晰得像是不同的人写的 |

**检查方法（两问，任一答"是"→ 触发）：**

1. **功能标签测试**：给每段贴一个结构功能标签（开场钩子 / 痛点展开 / 归因解释 / 重框 / CTA）。有没有两段以上的标签切换让读者感到"上一段还在说 X，怎么突然跳到 Y 了"？有 → 触发。

2. **删段测试**：删掉情绪转折段，前后文是断了吗？还是更顺了？如果删掉后读起来更自然 → 那个转折段是被结构要求逼出来的，不是思考需要 → 触发。

### 结构信号结果格式

```
□ 信号 1 · 结论可预测性：{通过 / 触发 → 依据}
□ 信号 2 · 中间转折：    {通过 / 触发 → 依据}
□ 信号 3 · 细节特异性：  {通过 / 触发 → 依据}
□ 信号 4 · 语言均匀度：  {通过 / 触发 → 依据}
□ 信号 5 · 结构挤压感：  {通过 / 触发 → 依据}

触发数：[X]/5
{触发 < 2 → 无模板生产嫌疑}
{触发 ≥ 2 → 有模板生产嫌疑 → 回到 STEP 5，优先解决信号指出的问题}
```

> ⚠️ 这不是在判断"这篇内容真诚不真诚"——AI 没有这个能力。这是在识别"这篇内容的结构是否暴露了它是被模板生产的"。最终的判断留给用户。

## 情绪基调判断（写作决策工具）

> 在进入校验二之前，先判断这篇的情绪基调属于哪种。这影响如何评估情绪弧线。

| 基调类型 | 定义 | 弧线特征 | 典型主题 |
|---------|------|---------|---------|
| **唤醒型** | 情绪被压抑，需要被看见和释放 | 需要上升弧线：被压住 → 愤怒 → 力量 | 身份撕裂、自我怀疑、愤怒被压抑、倦怠 |
| **陪伴型** | 情绪已被感知，需要被理解和接纳 | 弧线平缓合理：困惑 → 轻轻明白 | 孤独、失望、迷失、被误解、渴望自由 |

**使用方式：** 在 STEP 5 开始写作前判断基调（因此应在 STEP 6 校验开始前先回看确认）。这影响矛盾句的强度和结尾姿态——唤醒型可以更有力，陪伴型需要更轻。

  G1/G2（禁用词/结尾非处方）→ STEP 5（文案写作）
  G3（「」引出）→ STEP 3（隐喻挖掘）或 STEP 5（文案写作）
  G4（具体锚点）→ STEP 5（文案写作）
  G5（视角漂移）→ STEP 5（文案写作）
  G6（标题专项）→ STEP 5（文案写作）

全部通过 → 进入 STEP 7。
