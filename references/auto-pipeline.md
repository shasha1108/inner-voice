# 流程 E：自动选题入库

> 触发：`/inner-voice 自动选题`（定时任务专用 · 无人值守）
>
> 每天早上自动执行全域趋势调研，取 10 个方向直接写入飞书选题资产库，完成后发邮件汇总。

## 定时任务

**时间：** `references/cron-schedule`（1 行纯 cron 表达式，当前值见该文件）

**改时间：** 编辑 `cron-schedule` → 删除旧定时任务 → 用 CronCreate 重建。

## 执行规则

整个流程不询问用户、不等待确认。唯一通知出口 = Agent Mail 汇总邮件。

## 流程

### 1. 前置检查

```
opencli doctor 2>&1
→ 失败 → 发告警邮件「选题自动入库：opencli 离线，今日跳过」→ 流程结束
```

### 2. 执行 STEP 0（全域趋势调研）

按 SKILL.md STEP 0 流程执行（加载 trend-matrix.md → 7 步调研 → 产生候选方向）。
**不输出推荐卡片给用户。** 内部推演，只记录方向列表。

### 3. 筛选

取推荐指数最高的 10 个方向。不足 10 个 → 有多少取多少（最少 3 个，低于 3 个发告警邮件并跳过入库）。

### 4. 静默入库

对每个方向：
1. 按 `assets/templates/recommendation-card.md` 输出格式组装字段值
2. 组装 JSON payload — 字段定义 → Read `scripts/topic-upsert.sh`
3. 写入飞书 — 参数和错误处理见 STEP 9「飞书写入通用约定」，table-id 使用 `$FEISHU_TOPIC_TABLE_ID`

逐条写入，记录成功/失败条数。

### 5. 发送汇总邮件

用 `agently-cli message +send` 发邮件给 `zhangwensha@agent.qq.com`，**直接带 `--confirmation-token` 发送，不等待确认。**

主题：「选题自动入库 — YYYY-MM-DD」
正文格式：

```
今日自动存入 N 个选题方向：

1. [方向标题] · 搜索价值 [高/中/低] · 推荐指数 [1-5星]
2. [方向标题] · 搜索价值 [高/中/低] · 推荐指数 [1-5星]
...

失败：X 条（已跳过）
```

### 6. 失败处理

- opencli 不可用 → 告警邮件，跳过入库
- 方向不足 3 个 → 告警邮件，跳过入库
- 飞书写入部分失败 → 邮件标注失败条数，成功的正常入库
- Agent Mail 发送失败 → 静默（不留日志，下次定时任务会重新正常运行）
