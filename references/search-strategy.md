# 搜索优化策略

## ⚡ 速查卡（校验步骤只读此节）

**标题植词**：密语感 + ≥1个可搜索场景词；自然融入，不堆砌
**高频场景词库**：时间（深夜/失眠/凌晨/下班后）· 空间（独居/工位/通勤/出租屋）· 情绪（焦虑/崩溃/emo/内耗/迷茫/倦怠）· 关系（社恐/讨好型/i人）

**钩子CES公式**：具体场景锚点（最近一次/在什么场景下/第几个夜晚）+ 开放式回忆邀请 = 引导≥8字评论
✅好钩子："你最近一次觉得'算了'，是在什么场景下？"
❌差钩子："你也有过这种感觉吗？"

**标签4层结构**：
- 搜索入口词(2个，月搜索1w+)：#躺平自救 #焦虑自救 #情绪管理 #心理健康 等
- 垂类精准词(3个)：#神经系统调节 #内耗自救 #精神内耗 #敏感体质 等
- 长尾场景词(2个)：从文案具体场景提炼
- 形式标签(1-2个)：#情绪插画 #情感创作（禁止用#心理学插画，因正文无学术词汇对应）
**禁止**：不堆砌情感/成长/自我等无差异泛词；治愈不单独用

**标签-正文一致性（STEP 7硬门）**：每个标签的核心词必须在正文或标题中有对应语义出现；孤立标签（正文零相关词）必须替换，否则算法判为"标签党"

## 核心认知

小红书约40%+流量来自搜索入口。算法通过NLP技术捕取笔记标题、正文、话题的关键词，给笔记打标签，再将标签与用户搜索词做匹配。

**搜索优化不等于SEO堆砌。** 对 而言，搜索优化是"让正确的人找到你"——把已有的情绪洞察力，用可搜索的语言包装，让真正需要这些内容的用户通过搜索找到你。

## 标题植词规则

**原则：密语感 + 搜索词，两者并存**

标题必须同时满足：
1. 有过类似经历的人产生"这说的就是我"的击中感
2. 包含至少1个可搜索的场景词

**示例对比：**

| ❌ 纯密语（无搜索量） | ✅ 密语+搜索词 |
|---------------------|---------------|
| 允许自己，做一个脆弱的人 | 深夜崩溃的真相：你不是脆弱，是终于安全了 |
| 最深的困局：随时能出走，却选择留在原地 | 上班焦虑 ｜ 随时能辞职，却每天准时打卡的人 |
| 活在情绪里的自己 | 失眠的夜里，你的身体在替你说什么 |

**高频场景搜索词库（持续更新）：**

| 类别 | 高搜索量场景词 |
|------|--------------|
| 时间场景 | 深夜、失眠、凌晨、周一、下班后 |
| 空间场景 | 独居、合租、工位、通勤、出租屋 |
| 身份场景 | 打工人、应届生、考研、裸辞、自由职业 |
| 情绪场景 | 焦虑、崩溃、emo、内耗、迷茫、倦怠 |
| 关系场景 | 社恐、讨好型人格、i人、断联、冷暴力 |
| 身体场景 | 失眠、头痛、胸闷、没食欲、总是很累 |

## 正文搜索词植入

**原则：自然融入，不改变文案美感**

- 正文第一段（召唤段）包含1-2个场景化搜索词
- 植入方式：用具体场景替代抽象描述

| ❌ 抽象（不可搜索） | ✅ 场景化（可搜索） |
|-------------------|------------------|
| 某个疲惫的时刻 | 加班到凌晨两点的那个瞬间 |
| 情绪涌上来的感觉 | 独居的夜里突然哭出来 |
| 说不出口的累 | 通勤地铁上闭着眼不想睁开 |

## CES 评分优化设计

**背景：** 小红书CES评分体系中，评论字数≥8字且包含具体场景描述的互动，权重是普通评论的3倍。

**当前问题：** 的内容评论大多是"好真实""说到心坎上了"——短评论，缺乏深度互动。

**优化方向：** 通过钩子设计引导读者输出≥8字的具体场景回忆。

**钩子CES设计公式：**

```
高CES钩子 = 具体场景锚点 + 开放式回忆邀请

✅ 好钩子（引导具体回忆）：
"你最近一次觉得'算了'，是在什么场景下？"
"你上一次在深夜突然清醒，脑子里在想什么？"
"独居的第几个夜晚，你开始习惯和自己说话？"

❌ 差钩子（只能回复是/否/共鸣）：
"你也有过这种感觉吗？"
"你今天还好吗？"
"是不是说到你心坎上了？"
```

**关键区别：** 好钩子包含一个"场景锚点"（最近一次/在什么场景下/第几个夜晚），让读者不由自主地回忆一个具体画面，从而写出有场景、有细节的长评论。

## 🚨 标签搜索强制验证（硬性前置门 · STEP 7 第一步执行）

> **此验证为硬性门，未通过不得输出标签。**

### 验证流程

**对每个候选标签，必须执行以下三步，并输出验证结果：**

```
对每个候选标签：
1. 打开小红书APP → 搜索该标签（带#号） → 查看搜索结果页顶部显示的笔记数量
2. 检查该标签是否有"搜索建议词下拉"（输入标签关键词，看小红书是否自动补全）
3. 查看该标签下前3条笔记的数据（点赞/收藏），确认内容方向和本笔记一致
```

### 验证输出格式（必须包含在STEP 7输出中）

```
【标签搜索验证报告】

#标签A → 搜索结果：{X}篇笔记 → 搜索建议词下拉：{有/无} → 前3内容与本笔记方向：{一致/不一致}
#标签B → 搜索结果：{X}篇笔记 → 搜索建议词下拉：{有/无} → 前3内容与本笔记方向：{一致/不一致}
...

验证方式：[联网实时搜索 / 基于经验评估（需注明具体依据）]
```

### 硬性禁止规则

| 规则 | 处罚 |
|------|------|
| 搜索结果 < 100 篇笔记的标签 | **禁止作为搜索入口词**（可降级为长尾场景词） |
| 搜索结果 = 0 的标签 | **禁止使用**（完全删除，从标签矩阵中移除） |
| 无搜索建议词下拉的标签 | **禁止作为搜索入口词或垂类精准词**（平台标签库未收录） |
| 前3条内容方向与本笔记不一致的标签 | **警告**：该标签的用户意图与本内容不匹配，建议替换 |
| 未执行搜索验证直接输出标签 | **标签无效，必须回STEP 7重新验证** |

### 自造词检查

**你的标签里有创作者自己发明的词吗？**

检查方法：搜索该词 → 看搜索结果。如果前20条里没有与"情绪/心理/插画"相关的内容 → 这个词在平台不存在 → 删除。

> ⚠️ **历史教训**：「安静的倦怠」搜索量=0，两篇笔记浏览量均<20。自造词=零搜索入口=内容无法被发现。

## 标签搜索量分层

> 输出 15 个标签供用户筛选发布。最终发布时用户从中挑选，实际使用数量不限（小红书允许的标签数不固定）。

| 层级 | 月搜索量 | 数量 | 作用 | 选择原则 |
|------|---------|------|------|---------|
| 搜索入口词 | 1w+ | **2个** | 吃搜索流量 | 品类核心词，搜索结果前3条内容方向与本笔记一致 |
| 垂类精准词 | 1k-1w | **6个** | 精准匹配目标用户 | 从标题关键词、文案核心意象、品类衍生词中提炼 |
| 长尾场景词 | 100-1k | **5个** | 从文案提炼的具体场景 | 必须是文案中出现过的词或语义场景，零孤立标签 |
| 形式标签 | — | **2个** | 内容形式标识 + 系列感 | 1个账号视觉标识 + 1个系列/品类归属 |

### 双轨标签策略

| | 核心品类（情绪命名） | 氛围品类（梦核/既视感） |
|------|---------|---------|
| 入口词侧重 | 高搜索情绪词（精神内耗/情绪管理） | 品类核心词（梦核/既视感）+ 交叉词 |
| 垂类词侧重 | 心理学/疗愈/内耗自救类 | 美学/氛围/体验感类 |
| 长尾词侧重 | 具体场景（独居/职场/深夜） | 意象描述（似曾相识/记忆闪回） |
| 差异化锚点 | #情绪插画（vs 心理学插画） | #氛围感插画（vs AIGC梦核内容） |

**推荐高搜索量标签组合：**
- 核心：#情绪管理 #心理健康 #焦虑自救
- 长尾：#深夜emo怎么办 #上班焦虑 #敏感体质 #神经系统调节
- 场景：#独居日常 #职场情绪 #失眠自救
- 形式：#情绪插画 #心理学插画

**禁止：** 不得堆砌「情感」「成长」「自我」等无差异超泛词；「治愈」不单独用；禁用 #心理学插画（正文通常无学术词汇对应，导致标签-正文不一致）。

## 标签-正文一致性校验（STEP 7 硬性前置门）

**背景**：小红书算法通过NLP对笔记正文打标签，再和话题标签做交叉匹配。如果标签词在正文中完全没有相关词语或语义场景，系统会将该笔记降权标记为"标签党"（hashtag spammer），影响推荐分发。

**校验流程**：

```
对每一个准备使用的标签，检查：

□ 该标签的核心词在正文（标题+正文正文+钩子）中是否有对应词语或语义场景出现？

例：
#神经系统调节 → 正文必须出现"神经""系统""调节""神经系统""神经回路"等相关词语 ✅
#神经系统调节 → 正文只有"躺平""休眠""疲惫"等，无任何神经相关词 ❌ → 替换标签

处理方式：
→ 全部通过：进入STEP 7标签输出
→ 有标签孤立：先替换该标签为与正文内容相关的标签，再输出
→ 禁止反向操作：不得为了保留标签而修改已确认的文案
```

**形式标签建议替换**：
- ~~#心理学插画~~ → 改用 #情绪插画（正文是情绪叙事，不是心理学科普）
- #情感创作 → 当正文包含情感表达时可用
- #情绪图文 → 当正文情绪感强时可用

## 词条入库搜索验证（硬性前置门）

**规则：任何词条入库前必须在小红书搜索框验证。**

1. 打开小红书 → 搜索该词 → 看到下拉建议词 → 看"综合"页前 3 条笔记数据
2. **搜索量 0 的词条禁止入库**（即使内部分析再好）
3. **词条命名三选一**：① 用用户已用词（高检索词）② 在用户已用词基础上做轻量修辞 ③ 完全不强调命名（直接用场景词做标题）
4. **标签矩阵的底座必须是经过搜索验证的词**——自造词永远不能做底座

**Why:** 标签底座=搜索入口；入口词搜索量为 0 = 标签失效 = 内容无法被搜索到。

## 数据塌方检测与破冰模式

**塌方信号：** 连续 2 篇浏览 < 50 或 5 篇平均浏览 < 200

**塌方时处理：**
1. **同一方向连续 2 篇浏览 < 50 → 立即停止当前方向 30 天**
2. 停止期间：① 拆解 3 个同账号粉丝量级 + 题材相近的爆款 ② 重新做搜索验证 ③ 换破冰策略再启动
3. **不要在原方向上"再试一次"**——数据已经给了答案

**破冰模式（触发后强制）：**
- 标题用高搜索量用户原话
- 视觉用大字+实拍
- 钩子设计成 3 选 1（降低评论成本）
- 标签底座用搜索验证过的词
- **不破冰不恢复正常生产**

## 爆款基因年度校准

**规则：每季度必须做一次"爆款基因适配性检验"。**

1. 找当月小红书同领域爆款 5 个，分析它们的载体和呈现方式
2. **承载载体过时 → 改载体，不改基因**——隐喻本身还能用，但呈现方式要更新
3. 新词条入库时标注观察到的趋势特征，后续复用时重新验证
