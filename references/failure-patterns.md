
# 失败模式库

> 仅收录复盘确认质量差（点赞<20 且 收藏<10 且 评论<5）并完成诊断的内容。

## 使用说明

STEP 3 加载时：
1. 对照失败模式分类 → 检查当前隐喻/角度/视觉是否命中已知死区
2. 看失败案例中是否有与当前选题相似的 → 避开相同的坑
3. 不是"吓到不敢做"，是"做之前知道哪里会翻车"

## 记录规则

### 入库条件（全部满足）

1. Flow D 复盘确认质量差（数据全部低于红线）
2. 完成对照诊断（至少匹配到 1 个失败类型）
3. 提炼出可操作的避免方法

### 禁止入库

- 未诊断（只有数据没有分析）
- 失败原因是外部因素（限流/发布时间/账号权重）→ 不往这记
- 已在 SKILL.md 中固化的规则 → 直接修 SKILL.md，不重复记

**Why:** "点赞 3、收藏 0"不是失败模式，是失败结果。模式是"为什么会这样"——没有诊断就没有模式。

## 核心死亡陷阱

> 来自历史爆款（3480❤/1810❤）与失败新作（均≤5❤）的逐帧对比。

**「形式对了，魂没了」** — SOP 化复制了爆款的外壳，但遗漏了爆款的魂。

爆款的核心价值 = 一个在那个时刻、对那批用户、第一次被准确说出来的情绪隐喻。「漩涡」只爆一次，「玻璃盒」只爆一次。用相似的隐喻做相似内容，用户感受到的是「又是这种类型的帖子」。

每次创作的核心问题不是「这个情绪该用什么模板」，而是「这个情绪，有没有一个从没被人这样说过的角度？」

## 失败模式分类（诊断框架 · 12 类）

> 逐条对照诊断时使用。每条新案例挂载到对应类型下。

### 1. 封面情绪方向错位

| 表现 | 根因 |
|------|------|
| 主题"失控/想停下来"，封面图却是粉色梦幻海边走向地平线（=自由/希望） | 图的情绪信号和标题完全相反，0.5秒被划走 |

### 2. AI 图同质化

| 表现 | 根因 |
|------|------|
| 超现实山野+孤门+人影——已被大量账号复用 | 视觉稀缺性消失，无停滑力 |

### 3. 混入摄影图

| 表现 | 根因 |
|------|------|
| "情绪插画"账号封面用了真实摄影（暗黄雾夜桥上行走） | 破坏"原创插画创作者"的视觉标识 |

### 4. 文案先给概念再解释

| 表现 | 根因 |
|------|------|
| "我们可以把这个状态叫做'自动运转'。不是你变迟钝了，而是……" | 读者感受到被分析而非被陪伴 |

### 5. 标题描述状态而非触发认同

| 表现 | 根因 |
|------|------|
| "最深的困局：随时能出走，却选择留在了原地"（宏大词拉远距离） | 缺乏密语感，像内容介绍而非情绪触发 |

### 6. 主角透明化

| 表现 | 根因 |
|------|------|
| `transparent silhouette` / `hollow body` / `see-through figure` 作用于主角 | 鬼魂/泥浆质感 |

### 7. Magritte 式空洞 cutout

| 表现 | 根因 |
|------|------|
| 「人形镂空透过画布看见背后房间」——连续两套不同方向均得 2/10 | MJ 精度不足；概念层面根本错误，非 Prompt 写法问题 |

### 8. 图文不同隐喻

| 表现 | 根因 |
|------|------|
| 文案核心隐喻是「吞回去的不积累」，图片却切换到「镜中无倒影」 | 图文割裂，视觉没有从 confirmed_metaphor 出发 |

### 9. 配色忽视停滑力

| 表现 | 根因 |
|------|------|
| 「疲惫/疏离」→「低饱和灰蓝白系」，图片在信息流里隐形 | 情绪匹配 ≠ 信息流竞争力；暗调图在嘈杂信息流里无法被看见 |

### 10. 矛盾句强行拼凑

| 表现 | 根因 |
|------|------|
| 「不不」相邻 / 多重从句叠加 | 只做书面逻辑检验，没有逐句朗读检验 |

### 11. 叙事型隐喻伪装成空间型

| 表现 | 根因 |
|------|------|
| 地图蓝点/拆信/台词空白——需要叙事展开才能理解，单帧无意义 | 没有通过前置视觉主体性测试 |

**已验证的叙事型伪装案例：**

| 隐喻 | 表面理由 | 为什么失败 |
|------|---------|---------|
| 地图上蓝点消失 | "有空间形状" | 蓝点是屏幕像素，不是主体，画面只有"一个人看手机" |
| 拆信发现陌生人清单 | "有物理形状（信封）" | 需要叙事展开（寄信→等待→拆信→发现），单帧无法传达 |
| 台词空白（台上忘词） | "有舞台场景" | 需要知道剧场背景 + 理解"20年的角色"，单帧只是"一个人站在舞台上" |
| 影子滞后于本体 | "有空间关系" | 依赖观者主动校验物理细节，普通人不会注意影子的微小偏移 |
| 悬停的硬币 | "有凝固状态" | MJ 无法稳定生成悬停硬币；画面太小，不是主体级别 |
| 旧密码突然不对了 | "有体感" | 密码键盘是道具，画面是"一个人对着门锁"——叙事性的 |

### 12. 隐喻命名"情绪机制"而非"中心感受"

| 表现 | 根因 |
|------|------|
| 「讨好型疲劳」三版全败：命名"如何表演"，没有命名"不知道自己是谁了"的中心感受 | 读者经验三问只检验体感真实性，不检验是否命中中心感受 |

### 13. 物理动作与心理意象割裂（掩耳盗铃式伪共鸣）

| 表现 | 根因 |
|------|------|
| 大动作（捂耳/跪地/摔物/关灯）+「没人察觉」= 矛盾 | 物理强度大的动作在人群中必被看见；强行赋予隐秘感 = 物理不成立 |
| 走"实写"路径却要求"无人察觉" = 掩耳盗铃 | 隐喻没在两条合法路径（实写/虚写）间明确选一种 |
| 形式校验全过但读者读出违和感 | 现有质量门只检形式不检现实 = 系统性盲区（见案例 #2） |

### 14. 场景元素堆砌（假具体）

| 表现 | 根因 |
|------|------|
| 「手机在响 + 亲戚在问 + 同事在等回复」= 多个具体元素并列 | 同一时点 ≥ 3 个具体焦虑源 = 场景崩塌，读者脑内画面挤爆 |
| 看起来具体，实际是「罗列焦虑源」 | 真正的具体 = 1 主源 + 细节（动作/神态/环境音）；假具体 = N 个名词并列 |

### 15. 强行代表读者（"我们大多数人+非常规记忆"）

| 表现 | 根因 |
|------|------|
| 「我们大多数人，都是在被问到'你没事吧'的时候第一次捂住耳朵」 | 人第一次捂耳朵的真实记忆是鞭炮/雷声，不是心理触发；强行代表 = 潜意识抗拒 |
| 用「我们大多数人/很多人/大家」+ 并不普遍的具体经验 | 读者会立刻感到"我没有，你别代表我"，共鸣断裂 |

### 16. 形式校验全过但现实崩塌（系统性盲区）

| 表现 | 根因 |
|------|------|
| G1-G6 + 校验二/三 + 结构信号全部通过，但文案读出违和感 | 现有校验只检文字形式（词数/矛盾句/视角），不检物理现实（动作是否真实发生） |
| 一篇形式完美但场景崩塌的稿子能通过所有门 | 校验是必要条件不是充分条件；缺"现实自洽性"作为最高优先级门 |

## 三大隐喻挖掘陷阱

> 每次 STEP 3 隐喻挖掘前过一遍。

**字面翻译陷阱：** 从情绪词 → 物理形容词 → 找物体（崩溃→碎→杯子/地面）= 全是同一套损坏皮肤，无悖论。

**缺美感陷阱：** ONE MASSIVE 通过 ≠ 美。候选必须能说清"为什么这个画面是美的"。

**物理强度未明陷阱：** 候选隐喻只问"它能不能命名情绪"还不够，必须问"它的物理强度有多大"——大动作（捂耳/跪地/摔物）= 强制走实写路径（需要私密空间）；小动作（深呼吸/走神）= 强制走虚写路径（可发生在人群前）。混用 = 案例 #1 掩耳盗铃式伪共鸣。

### 案例 #1 · 2026-06-26 「边界感」— 掩耳盗铃式伪共鸣（物理强度与隐秘感矛盾）

**数据**：未发布（用户拦截，质量校验形式全过 0/5 但现实崩塌）

**诊断**：命中失败模式分类第 1 类（封面情绪方向错位）+ 新增 13 类「物理动作与心理意象割裂」+ 14 类「场景元素堆砌」+ 15 类「强行代表读者」

**根因**：
1. 「捂耳」是大动作（双手盖耳廓 = 视觉冲击大、社交显眼），物理上必须走"实写"路径——给私密空间（洗手间、关门的房间）
2. 但同时写「没一个人察觉」= 强行走"虚写"路径（隐秘感）
3. 两条路径在同一个动作上同时启动 = 物理强度（大动作）与隐秘感（无察觉）矛盾 = 掩耳盗铃
4. 「我们大多数人+第一次捂住耳朵」= 强行代表读者——人第一次捂耳朵的真实记忆是鞭炮/雷声，不是"被问你没事吧"
5. 「手机还在响+亲戚还在问+同事还在等回复」= 多个具体元素在同一时点并发 = 缝合怪场景 = 假具体
6. 触发词情绪动力学不一致：「你没事吧」（关心试探）→「我都是为你好」（道德说教）= 两种完全不同情绪语境被强行缝合

**教训**（可操作）：
- **G7 物理现实校验**：物理动作放进现实场景中可发生 = 必要条件；形式校验全过 ≠ 现实成立（见 `quality-signals.md` G7）
- **G8 隐喻路径二选一**：大动作（捂耳/跪地/摔物）→ 必须给私密空间 + 实写；小动作（深呼吸/走神）→ 必须可发生在人群前 + 虚写。**混用 = 必失败**
- **G9 场景单一深挖**：单一时点只 1 个具体物理切片写出厚度，禁止并列堆砌 ≥ 3 个具体焦虑源
- **触发词情绪动力学一致**：全文触发词必须同一种情绪动力学（要么关心试探/要么道德说教），不能跨段切换
- **避免「我们大多数人+非常规记忆」**：强行代表读者 = 潜意识抗拒 → 改"我"/删"第一次"/改"想"为心理活动

**修复方向**（用户提供的两个方案）：
- 方案 A：走实写——"关上洗手间门的那一刻，我第一次用双手死死捂住了耳朵"（挪空间）
- 方案 B：走虚写——"在对方那句话落下来的时候，我的听觉却突然像被抽空了。我安静地看着他们的嘴唇开合，只听见自己深沉的呼吸。没有一个人察觉到，我刚刚在心里捂住了耳朵"（改微动作+虚写）

### 案例 #2 · 2026-06-26 「边界感」— 形式校验全过但现实崩塌（系统性盲区）

**数据**：未发布（同案例 #1）

**诊断**：失败模式分类（新增 16 类）—「形式通过 ≠ 现实成立」的系统性盲区

**根因**：
1. 现有质量门 G1-G6 + 校验二/三 + 5 个结构信号全部是"形式校验"——检查词数、矛盾句、视角漂移、结尾姿态、模板生产嫌疑
2. **没有任何一项检查"物理现实"**：动作在真实场景里能否发生？空间是否自洽？多元素是否同时并发？
3. 因此一篇物理上崩塌（掩耳盗铃）的文案能轻松通过所有形式校验 = 表面精致但读者读出违和感
4. 根因层级：质量门的**设计盲区**——只检"文字形式是否像好文案"，不检"文字描述的世界是否真实"

**教训**（可操作，结构性修复）：
- **新增 G7 物理现实校验作为最高优先级**（已写入 `quality-signals.md`）：物理动作放进现实社交场景中可发生 = 必要条件；G1-G6 全过但 G7 失败 → 仍打回
- **新增 G8 隐喻路径二选一强制选择**（已写入 `quality-signals.md` + `writing-techniques.md` + `account-dna.md` + `SKILL.md` STEP 5）
- **新增 G9 场景单一深挖**（已写入 `quality-signals.md` + `writing-techniques.md` + `account-dna.md`）
- **质量校验顺序重排**：G7 → G8 → G9 → G1-G6。任何 G7-G9 失败 = 直接打回，不进入形式校验
- **根本教训**：写"看起来像好文案"和"读起来像真的发生"是两件完全不同的事。校验一二三 + 结构信号是必要条件，不是充分条件

### 案例 #3 · 2026-06-28 「边界感」— 视觉隐形（停滑力为零 + 无主体）

**数据**：点赞 0 / 收藏 1 / 评论 0

**诊断**：命中失败模式 9（配色忽视停滑力）+ 新增图案化问题（无叙事主体）

**根因**：
1. 全图灰色背景 + 浅灰脚印 + 白色粉笔线，无高对比度元素（仅左上角一根淡黄色粉笔笔头），在信息流中视觉隐形——不是看完不喜欢，是根本没被停住
2. 脚印均匀铺满全图形成重复纹理（wallpaper 效果），无叙事焦点；账号停滑力来自「强主体 + 情绪场景」，此图无主体可供眼睛落点

**注**：文案质量相对正常（隐喻转折「踩线最多的人是你」有力，结尾「粉笔在你手里，一直是」简洁），视觉问题预计占失败原因 90% 以上，在读者看到标题之前已划走。

**教训**：
- **情绪暗淡 ≠ 配色暗淡**：「边界被侵蚀」是沉的情绪，但封面必须在信息流中被看见——至少一个高对比度主体（人物/物件/文字），不能让情绪色调吃掉视觉竞争力
- **禁用 pattern/纹理铺满式构图**：均匀散布的重复元素（脚印/花/线条铺满）= 壁纸，不是插画；必须有一个明确的视觉主角占据画面 1/3 以上
