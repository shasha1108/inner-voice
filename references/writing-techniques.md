## 爆款文案结构（参考框架 · 以 account-dna.md 四步模型为准）

> ⚠️ 本节的"开头→中间→结尾"是通用参考框架。inner-voice 的核心结构模型是 account-dna.md 中的四步模型（召唤→命名→解构→重框），结尾姿态为接纳/重框而非行动指令。以下钩子类型和价值类型仍可作为 STEP 5 写作时的技法参考。

```
开头引入痛点 → 中间创造价值 → 结尾自然收束
```

| 环节 | 作用 | 占篇幅 |
|------|------|--------|
| **开头 · 3秒钩子** | 让读者停下。从下方「开头11种钩子类型」中至少选择 1 种 | 20% |
| **中间 · 创造价值** | 让读者觉得"这篇值得看完"。从下方「4种价值类型」中至少提供 1 种 | 60% |
| **结尾 · 收束** | 接纳/重框姿态，不给行动建议。CTA 自然即可，不强制 | 20% |

## 开头 · 11种黄金3秒钩子类型

> 写完后检查：我的开头属于以下哪种？如果没有一种对得上 → 开头太弱，重写。
> 前 10 种为基础钩子，第 11 种「复合钩子」为进阶叠加用法。

| # | 类型 | 示例 | 适用场景 |
|---|------|------|---------|
| 1 | **问句式** | 「周末躺了一整天，为什么比上班还累？」 | 戳中一个普遍困惑，让读者带着问题往下读 |
| 2 | **反转式** | 「我以前以为分手是因为不爱了。后来才想通——是没有力气继续假装自己没事了。」 | 从一个普遍认知出发，翻转它，读者被"那后来怎么了"钩住 |
| 3 | **场景式** | 「挂了电话，把手机扣在沙发上。没有发生什么特别的事。就是胸口很闷。」 | 一个具体的物理时刻，让读者立刻看见画面 |
| 4 | **爆款式** | 「翻了一下聊天记录。过去一周，我说了三十七次'没事'。没有一次是真的。」 | 用一个具体数字或事实砸中读者，制造冲击 |
| 5 | **观点式** | 「有一种累，不是做太多了。是一直在表演'不累'。」 | 直接抛出一个洞察，读者如果被击中就会停下来 |
| 6 | **反常识式** | 「你以为那是懒。但每一次什么都不想做的下午，可能是你的身体在替你拒绝一个你不想要的生活。」 | 把读者默认的自我判断翻转过来，制造认知缺口 |
| 7 | **数据式** | 「去年发了四百多条消息说好的。其中大概三百条，我本来想说的是别的东西。」 | 让数字替故事说话，读者会开始算自己的 |
| 8 | **总结式** | 「三十岁之后慢慢意识到：成年人的每一天，都在为十八岁做的决定买单。」 | 把一个时代共同的体验压缩成一句，读者想截图 |
| 9 | **评论式** | 「朋友说我的朋友圈越来越少了。我想了想，不是没东西发——是发了也没人真的在看。」 | 从一个日常观察展开，读者跟着一起往下想 |
| 10 | **颠覆式** | 「孤独不是一个人待着。孤独是和一桌人吃饭，但你不能让任何人看出来你今天不对。」 | 重新定义一个大家都以为自己知道的词 |
| 11 | **复合钩子** | 「早高峰的地铁上，周围全是人。我忽然想——这里除了我，还有谁在认真呼吸吗。」——场景式（早高峰地铁）制造即刻画面 + 反转式（「认真呼吸」把日常行为陌生化），两个钩子叠加 | 需要最强停滑力时使用：反转制造悬念，场景制造画面，同时触发 |

## 中间 · 4种价值类型

> 写完检查：读者读完中间部分，获得了什么？如果什么都答不上 → 这篇没有价值。

| 类型 | 定义 | 示例 |
|------|------|------|
| **知识价值** | 颠覆认知的理论 / 不为人知的真相 / 内行才知道的规律 | 「为什么周日晚上比周一早上更焦虑？焦虑不来自工作本身——来自你提前三天在脑子里把它做了一遍。」 |
| **情绪价值** | 被理解、被看见、被命名——「对，我就是这种感觉」 | 痛点挖掘5步（见下方） |
| **审美价值** | 视觉冲击力、构图、配色——让人想保存的图 | ONE MASSIVE + 悖论视觉化 |
| **社交价值** | 让人想转发/截图发朋友——「这个说的就是你」 | 可引用的金句、可截图的短段落 |

## 中间 · 痛点挖掘 5 步（情绪价值型内容核心技法）

```
发现焦虑 → 放大痛苦 → 揭露普遍性 → 展示严重性 → 总结痛苦代价
```

| 步骤 | 做法 | 示例（以「上班上到面无表情」为主题） |
|------|------|-------------|
| 1. **发现焦虑** | 命名——读者知道有事但说不清 | 「下班熄了火，在车里坐十分钟才上楼。不是累——是这十分钟里，你不需要对任何人负责。」 |
| 2. **放大痛苦** | 具象化——让读者身体感受到 | 「上个月用了八天年假。五天在医院排队，两天帮朋友搬家，剩下一天在家躺着。哪也没去。」 |
| 3. **揭露普遍性** | 「你不是一个人」——但不要说这句话，用共鸣让ta自己感觉到 | 「茶水间碰到的同事都说还行吧。你看了一眼他们的脸——没说出口的话比说出来的多。」 |
| 4. **展示严重性** | 「如果不改变会怎样」——用具体后果 | 「明年这个时候你还会在这个工位上，同一个时间，同一个感觉。只是又老了一岁。」 |
| 5. **总结代价** | 给整件事一个名字——让读者记住 | 「你每天八小时换来的工资，刚好够付周末把自己修好的费用。这笔账从来没有平过。」 |

## 结尾 · CTA 行动指令 + 互动触发器设计

**CTA 原则上应存在，但有例外。** 社会热议话题（有天然讨论属性）+ 结尾已给出完整观点 → CTA 可省略。好内容自己引发讨论。强行 CTA（读者本想说但被格式化成答案）不如没有。

**CTA 质量标准：**

好的 CTA 让读者想留下点什么。判断标准是看评论区的实际回复——全是「好的」「拿图」「1」= CTA 失败了。合格的 CTA 触发的是具体回应：一段自己的经历、一个具体的答案、或者一个读者和内容之间的暗号（只有当你文中的核心隐喻已经建立了、读者不需要你解释那个符号是什么意思时，符号 CTA 才有效）。

如果 CTA 需要解释自己的符号——「请在评论区留一个⭕，这代表你画的那个圈」——说明这个 CTA 还不成立。不要解释。解释 = 不信任读者已经懂了 = 读者确实还没懂。

**🚨 CES 权重背景（决定为什么评论比收藏重要）**

> 小红书 CES 公式：点赞×1 + 收藏×1 + **评论×4** + 转发×4 + **关注×8**
>
> 情绪插画的系统性陷阱：内容有治愈感 → 大量默默收藏（1分/个）+ 极少评论（4分/个）→ CES 严重偏低 → 算法停止推流。
> **1条评论 = 4个收藏的推荐权重。** CTA 的设计目标只有一个：让读者开口说话，不是让读者默默收藏。

### 4 种互动触发器（按 CES 贡献排序）

| 排序 | 类型 | CES贡献 | 机制 | 示例 |
|------|------|---------|------|------|
| 🥇 | **连载钩子式** | 关注×8（最高） | 留悬念 + 引导关注 | 「下一张画的是凌晨3点突然想通的一件事，先关注，这周发」 |
| 🥈 | **直接提问式** | 评论×4 | 结尾开放性问题 | 「你上一次莫名其妙就崩了——是哪一天？」 |
| 🥉 | **话题自激发式** | 评论×4（但依赖内容质量） | 矛盾描述 + 不直接提问 | 全篇矛盾句自然收尾——内容足够好时读者自己会想说 |
| 4 | **场景认领式** | 评论×4（短评为主） | 列举场景 + 编号暗示 | 正文中 3-7 个具象场景，「第3个太真实了！」 |
| 5 | **无提问·视觉主导式** | 收藏×1 为主 | 极简文案 + 治愈插画 | 3 句金句 + 高质量视觉——只适用于氛围品类，无法拉动推荐 |

**⭐ 最高可靠性评论触发格式（直接提问式的子类型 · 优先使用这四种）**

| 格式 | 示例 | 为什么有效 |
|------|------|-----------|
| **A/B 选择题** | "你遇到这种时候，是A哭出来，还是B假装没事？" | 1秒可回答，门槛极低，评论转化率最高 |
| **具体场景认领** | "今年有没有哪一天，你莫名其妙就崩了——是哪天？" | 触发个人记忆，读者有具体的话想说 |
| **开放式邀请** | "评论区说说最近最难受的是什么——我可能把它画下来" | 建立深度连接，评论质量高，黏性强 |
| **连载式关注钩** | "下一张画这个，先关注，别错过" | 唯一能触发关注（×8）的 CTA 格式 |

> **🚫 禁止使用的无效 CTA 格式：**"希望大家喜欢" / "觉得有用就点赞" / "有共鸣的话收藏备用" ——这类收尾不触发评论，CES 得分等同于没有 CTA，且会让算法认为帖子缺乏互动基础。

### 4 种 CTA 动作类型

| 类型 | 目的 | 示例 |
|------|------|------|
| **点赞型** | 「如果被说中了」—— 低门槛 | 「如果这篇让你想起那个在工位上突然想哭的下午——点一下。」 |
| **收藏型** | 「以后可能用到」—— 适合有金句 | 「存下来。哪天又觉得『是不是只有我这样』的时候，翻出来看一遍。」 |
| **评论型** | 引导具体回答 —— 最高互动权重 | 「你最近一次觉得这不是我想要的生活——是在哪个瞬间？」 |
| **转发型** | 「有人需要看到这个」 | 「转发给那个看起来没事、但你知道最近很难的朋友。」 |

## 对立修辞 · 4 种类型（矛盾句技法）

> 矛盾句不是「还差一组」逼出来的。选择适合本篇的类型，从思考中自然产生。

| 类型 | 公式 | 示例 |
|------|------|------|
| **情感矛盾** | 渴望X，却又Y | 「想被看见，又怕被看见之后对方觉得不过如此。」 |
| **行为-感受矛盾** | 明明A，却感觉B | 「嘴上说着没事。胃已经疼了三天了。」 |
| **认知矛盾** | 当我觉得X → 其实只是Y | 「我以为自己是不想社交。后来发现——是不想再演一次那个很会社交的自己。」 |
| **哲学悖论** | 重新定义一个词 | 「有时候最累的不是一直在走。是不知道往哪走，但不敢停。」 |

**使用要求：** 宁少勿多，宁心理型勿修辞型。叙事型全文 2-4 组，钩子型 3-5 组，超出上限 → 互相稀释。

> **修辞型矛盾 vs 心理型矛盾（核心区别）：**
>
> 修辞型是 A 和非 A 的对置，结构清晰，读者感到"对，这很矛盾"——容易写，但力量有限。
> 心理型暴露一个读者自己还没意识到的内部状态，令人不舒服，但令人信服：
>
> ❌ 修辞型："想被看见，又怕被看见之后什么都不同了。"（对置干净，读者点头，但不沉默）
> ✅ 心理型："我告诉自己没关系，然后我很生气——因为又有人告诉我没关系了，而那个人是我自己。"（暴露自我矛盾，读者沉默两秒）
>
> 写矛盾句前问：这组矛盾会让读者沉默两秒，还是只会点头？只是点头 → 是修辞型，可以保留但不要依赖。沉默两秒 → 是心理型，保留，这是这篇文案最值钱的地方。

**🚨 冗余对立检查（每组矛盾句必须通过）：**

```
□ 删掉这组对立修辞，前后文是否受影响？
 → 不受影响 = 它是多余的。去掉它。
 → 受影响（逻辑断裂/情绪断层） = 它是有用的。保留它。

❌ "手是抖的、把背伸直、把气喘匀"——三个身体动作堆在一起。删掉→前后文完全不受影响
✅ "那句话说出口之前，我以为会很难。结果没有。"——删掉→情绪弧线断裂
```

## 清单式共鸣场景公式

**公式：场景 = 时间/地点 + 具体动作 + 内心感受**

> 通过 3-7 个超具体的微观场景，让不同用户都能找到至少 1 个共鸣点。太少共鸣面窄，太多造成疲劳。

```
✅ "站在人群中却感觉在下沉"（地点：人群中 + 感受：下沉）
✅ "凌晨3点反复刷短视频的大拇指"（时间：凌晨3点 + 动作：刷手机 + 身体部位：拇指）
✅ "工位前删了又打的辞职信"（地点：工位 + 动作：删了又打）
```

**评论区激发机制：** 用户会评论"第3个太真实了！"→ 形成场景认领互动模式。

## 重新框架 · 3 种技法

> 将用户认为的缺陷/问题重新定义为正常/保护/必要。"重框"不是一个步骤名——是三种具体操作。

| 技法 | 公式 | 示例 |
|------|------|------|
| **正常化** | "X%的人都经历过" | 暗示：你的问题很普遍，不是你的错 |
| **功能化** | "距离感=保护机制" | 暗示：你的防御不是冷漠，而是自我保护 |
| **必要化** | "负面也是重要拼图" | 暗示：没有负面就不完整，它们是必需的 |

**可复用句式：**
```
X不需要被Y（治愈不是消除，而是共存）
X或许是Y（这不是麻木，是你的身体在保护你）
X也是重要的Y（那些裂缝，是光照进来的地方）
```

**🚨 重框的时机和确定性比内容更重要：**

| 重框类型 | 读者感受 | 使用建议 |
|---------|---------|---------|
| **确定型**："这不是你的错，你的身体在保护你" | 被轻微管理了 | 慎用；尤其忌讳在文案中段出现；出现太早 = 读者还没充分感受就被给了答案 |
| **脆弱型**："也许是……我不确定……但它一直在这里" | 被真正理解了 | 优先使用；不确定性本身就是陪伴 |
| **不重框**：停留在感受里，不给解释 | 深度陪伴 | 当情绪已经足够重，这往往是最有力的选择；"我只是很累了"比任何重框都有力量 |

上述三种技法（正常化/功能化/必要化）都是确定型。使用时必须放在文案下半段，且优先转化为脆弱型表达——把"距离感=保护机制"改成"也许那段距离，是你当时唯一能给自己的东西"。

## 隐喻用法一致性检查（写完每段后强制执行）

同一个隐喻在全文中必须用同一种方式使用。不能前半段当作意象/概念用，后半段当作物理空间用。也不能把隐喻用在不匹配的动词上。

```
逐句过当前隐喻。每一句里的隐喻是什么用法？
□ 意象/概念？→ "活成了一个谁都可以进来的地方"——"活成"修饰的是状态（活成了自己讨厌的样子），"地方"是空间。动词和宾语不匹配 ❌
□ 物理空间？→ "站在那个发光的圆圈里" ✅（圈始终是空间）
→ 两种用法出现在同一篇里 = 隐喻崩溃。动词和宾语不match = 文字堆砌。

正确做法（选一种，全篇一致。动词和宾语必须匹配）：
✅ "我们活成了一个不会拒绝的人"（"活成" + "人" = match）
✅ "我们把生活变成了一个谁都可以进来的地方"（"生活" + "变成地方" = match）
❌ "我们活成了一个谁都可以进来的地方"（"活成" + "地方" = mismatch——这是堆砌）
```

> **教训**："每画一次，圈外面的人少一两个。我们以为这是变小了。圈是为了让我们在里面能把背伸直。"——前两句圈是概念（人少=边界收紧），第三句圈是房间（在里面做动作）。读者在两句话之间要完成一次"你怎么跑进去了？"的空间跳跃。

## 隐喻路径二选一（最高优先级 · 写作前必做预声明）

> **最高法则：实写或虚写，二选一走到底，全文不可混用。** 混用 = 物理动作与心理意象割裂 = 掩耳盗铃式伪共鸣。

### 两条合法路径

| 路径 | 含义 | 必须满足的条件 | 典型动作 |
|------|------|--------------|---------|
| **A · 完全物理实写** | 真实发生的物理动作 | **场景必须在私密空间**（厕所、独处的车内、关门的房间），不允许在众目睽睽之下做奇怪动作而不被发现 | 双手捂耳、关灯、关上门、摔门 |
| **B · 完全心理虚写** | 心理活动外化 | **表面动作必须极其微小、符合社交常规**（深呼吸、短暂走神），"隐喻动作"只能在心里发生 | 深呼吸、走神、揉太阳穴、握拳、闭眼 |

### 写作前预声明（强制 · STEP 5 第一步）

> 在开始写作前，先在内部推演时明确告知本次隐喻的属性：
>
> **选项 A【完全物理实写】**：场景在私密空间，物理动作真实发生，旁人无观众。
> **选项 B【完全心理虚写】**：表面动作符合社交常规，隐喻动作只能在心里发生。
>
> **严禁在同一篇文章中混用物理动作与心理隐喻。**

### 物理强度判定（写完每段后自检）

```
□ 这个动作在现实里有多显眼？
 → 大动作（捂耳、跪地、摔物、关灯、撕纸）→ 必须走 A 实写
 → 小动作（深呼吸、走神、揉太阳穴、握拳）→ 必须走 B 虚写

□ 混用会怎样？
 → 大动作 + 「没人察觉」= 掩耳盗铃（动作大必被看见）
 → 小动作 + 「私密空间」= 多余（人群前场景已经够私密）
```

### 反例（2026-06-25「边界感」捂耳案例 · 形式完美但现实崩塌）

```
❌ "我们大多数人，都是在被问到'你没事吧'的时候第一次捂住耳朵的。
    抬起手，掌心贴上耳廓。
    松开手，发现没有一个人察觉到我们刚才在挣扎。"

失败原因：
  1. 捂耳 = 大动作 → 必须走 A 实写（需要私密空间）
  2. 但同时「没一个人察觉」= 强行 B 虚写（隐秘感）
  3. 混用 = 读者脑内画面"亲戚面前突然双手捂耳5秒，亲戚像瞎了一样继续说"
  4. 物理强度（大动作）与隐秘感（无察觉）矛盾 = 掩耳盗铃
```

### 两个修复方向

```
A. 走实写：把空间挪到私密处
  "关上洗手间门的那一刻，我第一次用双手死死捂住了耳朵。
   门外，手机可能还在震动，饭桌上的亲戚还在高声谈论。
   但那一秒，世界被掌心隔绝。"

B. 走虚写：把动作改成微动作
  "在对方那句'我都是为你好'落下来的时候，我没有反驳。
   我的听觉却突然像被抽空了。
   我安静地看着他们的嘴唇开合，只听见自己深沉的呼吸。
   没有一个人察觉到，我刚刚在心里捂住了耳朵。"
```

## 场景单一深挖（避免"焦虑源堆砌"式假具体）

> **核心：单一时点 1 个具体物理切片，写出厚度，禁止报菜名式堆砌。** 看起来"具体"≠真具体；真具体 = 1 个主源 + 细节（动作/神态/环境音），假具体 = N 个名词并列。

### 单一时点上限

```
✅ 单一深挖（1 个切片 + 细节）：
  "只是看着手机屏幕上，对方正在输入的省略号一闪一闪"
  "听对方说完那句'我都是为你好'之后，喉咙里那个'好'字卡了 3 秒"
  → 1 个具体物理切片 → 真实

❌ 报菜名堆砌（≥3 个具体元素并发）：
  "手机在响 + 亲戚在问 + 同事在等回复 + 朋友在催"
  → 多个具体元素在同一个时点并发 = 缝合怪场景
  → 读者脑内画面：面前一个亲戚 + 手机狂响 + 惦记同事 + 朋友催
  → 极度拥挤、不真实 = 情绪素材库拼凑感
```

### 判定三问

```
□ 我列了几个具体元素（人/物/事）？
  → ≤ 1 = ✅ 真实
  → 2-3 = ⚠️ 检查：主源 1 个 + 其他用泛指（"可能还在"）
  → ≥ 4 个具体并列 = ❌ 场景崩塌

□ 每个元素读者能在脑内画出独立画面吗？
  → 能 = 都在抢占读者注意力 = 互相稀释
  → 不能（泛指）= 让位给主源 ✅

□ 把场景压缩成一句话：这一刻，X 发生了。
  → X 是 1 个具体事 = 真实
  → X 是 "X+Y+Z 一起发生" = 堆砌
```

### 单一时点只 1 主源 + 泛指铺开示例

```
✅ "门外的世界还在响。
   手机可能还在震动，亲戚可能还在高声谈论。
   但那一秒，世界被掌心隔绝。"

❌ "手机在响，亲戚在问，同事在等回复"
  ← 3 个名词并列，无细节
```

## "强行代表读者"陷阱（避免"我们大多数人+非常规记忆"）

> **规则：用"我/我们"叙述具体经验时，确保这是大多数读者**真有过**的记忆。** 不真有的记忆 = 读者下意识抗拒："我没有，你别代表我"。

```
❌ "我们大多数人，都是在被问到'你没事吧'的时候第一次捂住耳朵的。"
  第一次捂耳朵的真实记忆 = 鞭炮声 / 雷声 / 工地噪音
  强行把它代表成"心理触发" = 读者抗拒

✅ 改为不涉及"第一次"的具体经验：
  "对方那句'我都是为你好'落下来的时候，我想捂住耳朵。"
  → "想" = 心理活动，不是物理动作 = 不强行代表
  → 或 "我试过捂住耳朵" = 单一第一人称，读者有共鸣或没有，不强制

判定：
□ 这个"我们大多数人"是事实吗？（大多数人真的这样？）
□ 如果不是事实 → 改"我" / 改为不涉及"第一次" / 删除
```
