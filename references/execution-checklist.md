# 执行流程图 + 文件加载清单

> 每次创作开始时加载本文件。每进入一个 STEP，对照下方表格确认对应知识已提取。

## 全局流程图

```
用户输入
 ├─ /推荐 ──→ STEP 0（趋势感知）──→ 用户选方向 ──→ STEP 1
 ├─ /<词> <类型> ──→ STEP 1
 └─ /复盘 <关键词> ──→ STEP 11

STEP 1 解析输入 + 受众锚定
STEP 2 情绪-媒介匹配 + 搜索价值预判（时间型→告警，用户选出路）
STEP 3 隐喻挖掘（内部推演，只展示候选+终选）
 ★ 人工确认点 A-1（隐喻确认）
STEP 4 叙事角度卡片
 ★ 人工确认点 A-2（角度确认）

┌─── STEP 5-9 内部静默执行 ─────────────────────────────┐
│    │
│ 🚫 此区间内所有产出（文案/Brief/标签）不得在评分通过前 │
│ 以任何形式展示给用户。唯一出口 = 评分通过后的确认点B。│
│    │
│ STEP 5 文案生成   │
│ └→ 完成后立即触发 STEP 8-9（不可跳过） │
│ STEP 6 视觉 Brief（可在评分后补）  │
│ STEP 7 标签生成（可在评分后补）  │
│ STEP 8-9 评分+阈值判断（标题≥9.0/正文≥9.5，最多2轮） │
│ └→ 不通过 → 静默修改 → 重评（不给用户看中间版本） │
│ └→ loop≥2 → 强制通过，附未通过原因  │
│    │
└────────────────────────────────────────────────────────┘

 ★ 人工确认点 B（最终输出 → 持久化）
 └→ 必须包含：评分报告 + 标题 + 正文 + MJ Prompts + 标签
 └→ 缺少任一项（尤其评分报告）= 流程无效，回 STEP 8-9

STEP 10 持久化（资产库 → git → 飞书）
```
