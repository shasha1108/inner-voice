# 配文模式

> **加载时机：** `/inner-voice 配文 <图片路径> [可选主题提示]`
> **此模式职责：** 图已完成，只需文案。不需要隐喻挖掘（图就是隐喻的视觉结果），不需要视觉 Brief（图已经有了），不需要为图评分（图不改了）。只需要把图里的情绪翻译成文字。

## 配文流程

### P0. 路径验证（先于一切 · 失败则终止）

使用 Read 工具确认路径存在且包含图片文件（PNG/JPG/WEBP）：
- 路径不存在 → 立即告知用户："路径 `[路径]` 不存在，请确认后重试。" → 终止。
- 目录为空或无图片文件 → 告知用户："目录 `[路径]` 内未找到图片文件（PNG/JPG/WEBP），请确认后重试。" → 终止。
- 不要在读图失败后继续生成文案——图片解读失败等价于无输入，继续执行会产生幻觉。

### P1. 读取图片

从 P0 验证通过的路径读取图片文件。如果路径是一个文件夹，读取文件夹内所有图片。支持的格式：PNG、JPG、WEBP。

### P2. 图像解读（内部执行，向用户输出 1-3 行摘要）

从图片中提取以下信息：
- 这张图的**情绪是什么**？（不是描述画面内容，是画面传达的感受）
- 画面里有**悖论**吗？（两种对立状态同时在画面里成立？是什么？）
- 如果是你（ ）画的，你想通过这张图让读者**感受到什么**？
- 图里有没有一个**可以被命名的东西**？（一个具象元素可以成为「」概念）

**🚨 设计选择 ≠ 隐喻内容（解读前强制判断）：**

```
□ 图里的颜色/材质/效果——是"它真的是这个东西"，还是"为了让它在画面里被看见"？
 → 橘色的线条，是真的火？还是为了让边界在暗色背景上可见？
 → 设计选择不能作为隐喻解读的依据。线条就是线条。圈就是圈。
 → 把"可见性设计"当成"隐喻内容"= 脑补投射 = 全篇建立在错误解读上
```

> **教训**：橘色发光线条+暗色背景 → 解读为"火焰圈"，整个文案建立在"火"的隐喻上。实际上那就是一个圈——橘色只是为了在暗色背景上能被看见。把设计选择当成了内容。

输出摘要：
```
图像解读：情绪=[...] ｜ 悖论=[...] ｜ 可命名的东西=[...]
```

### P3. 文案生成

> 📖 语感参照 → Read `references/exemplars.md`

从 P2 的解读出发写文案。图文必须一致——文案说的就是图里有的情绪，不对图做额外解释，不加图里没有的信息。

标题必须可搜索（含≥1个场景词）。正文用"我"或"你"视角——如果图里有人物，优先用人物视角；如果图是空景/物体，用"我"的观察视角。

**🚨 标题写作铁律（全模式通用）：**

正确的创作链路：

```
第一步：这篇文章最想说的那句话是什么？
 → 写出来。不思考维度、不思考搜索、不思考格式。就是一句话。

第二步：这句话里有可搜索的词吗？
 → 有 → 进入第三步
 → 没有 → 能不能让搜索词是句子自然的一部分？（不是贴标签）
 → 加不进去 → 不加。读起来像人话优先于搜索优化

第三步：这句话和图有关联吗？
 → 读者看到标题+封面，能不能立刻把两者连起来？
 → 不能 → 调整，但调整的必须是同一句话，不能跳到一个新句子

第四步：读出来。
 → 这是人在说话吗？→ 不是 → 作废，回第一步
 → 是 → 提交
```

**禁止：**
- 把维度当构造工具（"搜索性最高的词+图片最贴的词+结构最对的词"拼起来）
- 用«»书名号做标签装饰（已过时）
- 竖线前后是两个独立句子（"A｜B"必须能读成"AB"的连续关系，否则用单句）
- 为了匹配某个维度而用一个语义上不成立的词（"亮"不修饰"话"）

**如果只能记住一条：标题是你在说话，不是你在优化。**

**🚨 标题输出前强制自审（5问 · 全模式通用 · 一问不过=不发）：**

```
□ 1. 搜索词检查：这个标题里有至少一个可以被搜索的词吗？
 → 没有 = 不发。搜索词必须是句子自然的一部分，不是塞进去的
 → 🚨 声称"X是长尾搜索词/SEO友好"必须有小红书搜索验证数据，禁止凭空编造

□ 2. 人话检查：大声读出来——这是人在说话吗？
 → "好话。就是太亮了。"（亮不修饰话）→ 不发
 → "«想开点» ｜ 这张笑脸，不是我的"（前后断裂）→ 不发

□ 3. 角度一致性检查：标题的基调是否匹配 confirmed_angle？
 → D型归因="说不不是冷漠，是把门从里面关好"（归因+重框）
 → B型反问="我们被教怎么画大，没人教怎么画圈"（反常识）
 → 基调不匹配 = 不发

□ 4. 隐喻植入检查：标题是否包含隐喻核心词？（圈/漩涡/玻璃盒）
 → 包含 = ✅ | 不包含且隐喻已确认 = 不发

□ 5. 图片匹配检查：读者看到标题+封面，能立刻把两者连起来吗？
 → 标题说"圈"，封面是发光圆环 → ✅
 → 标题说"线"，封面是圆环 → 不匹配 = 不发
```

**违反任一条 → 不发，回到标题第一步重想。** 不需要用户来指出标题不好——这个自审应该在模型内部就拦截掉。
