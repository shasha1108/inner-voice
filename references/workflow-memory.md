> inner-voice skill 的运行时学习日志。记录创作过程中发现的新规律和失败诊断。
> **不写 project-level memory，不影响其他 skill。**

---

## 记录协议

### 什么值得记录

| 情况 | 记录 | 不记录 |
|------|------|--------|
| 用户否决了一个方向 | 诊断：为什么这个方向失败了 | "用户不喜欢X风格"（太模糊） |
| 用户确认了某个选择 | 成功要素：哪个决策被验证有效 | "这次写得不错"（无信息量） |
| 创作中发现新规律 | 具体规律+案例 | "应该注意X"（未经验证） |
| 数据复盘发现 | 数据+根因+可执行规则 | "下次要多关注X"（无操作性） |

### 什么不值得记录

- 已经在 SKILL.md 或 references/ 中明确写过的规则（去那里维护，这里不重复）
- 泛化的经验总结（"金句要密""视觉要强"——没有新的案例或精度提升）
- 纯猜测（没有实际数据或用户反馈支撑）
- 单次巧合（只发生一次、没有模式信号的事件）

### 写入格式

```
### [YYYY-MM-DD] [主题] — [一句话结论]
**事实**：[具体发生了什么]
**根因**：[为什么]
**规则**：[由此提炼的可执行规则]
```

### 维护规则

- **本文件上限 150 行**。超过时，将最旧的条目移到 `references/memory-archive.md`
- 已稳定且被多处引用的规则 → 提升到对应的 references/ 文件（account-dna / failure-patterns / search-strategy / visual-guide），然后从这里删除
- 每次创作时只读本文件最近 20 条记录（按日期倒序）

## 工作流协作铁律

1. **隐喻先确认再进叙事角度**（STEP 3 A-1 → STEP 4 A-2，不可跳过）
2. **视觉和文案用同一个隐喻**（STEP 6 从 confirmed_metaphor 出发，禁止图文不同隐喻）
3. **配色以停滑力为第一优先级**（先问"信息流里会被看见吗"，再问"和情绪匹配吗"）
4. **文案逐句朗读检验**（写完通读，不拗口、相邻无同音、句间有内在逻辑）
5. **全文只一个隐喻域，禁止域间漂移**（选定隐喻后列出词汇边界，全文不越界）
6. **多版失败后先质疑情绪-媒介适配性**（而非继续换隐喻）
7. **用户给的素材优先级 > skill 默认流程**（用户有明确方向时，不用默认流程）
