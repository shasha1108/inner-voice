
# 工作流记忆

> 创作过程中随手记的经验便签。轻量、临时、必读。

---

## 记录规则

### 记什么

| 情况 | 记录 | 不记录 |
|------|------|--------|
| 用户否决了一个方向 | 具体原因（"XX隐喻被否，用户说太抽象"） | "用户不喜欢X风格"（太模糊） |
| 用户连续做出同类选择 | 偏好信号（"今天连续两次选B型角度"） | "用户喜欢B型"（无上下文） |
| 创作中发现新规律 | 规律 + 具体案例（"XX句式在本篇情绪中失效，因为..."） | "应该注意XX"（无案例支撑） |
| 同一条规律被反复验证 | 标注验证次数，准备提升 | — |

### 不记什么

- 已在 SKILL.md 或 account-dna.md 中固化的规则
- 泛化经验（"视觉要强""金句要密"——没有新信息量）
- 纯猜测（没有实际数据或用户反馈支撑）
- 单次巧合（只发生一次、没有模式信号）
- 复盘结论（属于 success-patterns.md / failure-patterns.md）

**Why:** 便签不是文档——是草稿纸。写得越随意越容易记。格式太重 = 不想记 = 白设。

---

## 格式

一行一条，不超过 80 字：

```
[YYYY-MM-DD] [触发场景] → [发现]
```

示例：
```
2026-06-25 隐喻确认被否 → 用户说"漩涡"已被用烂，需要全新物理域的隐喻
2026-06-25 角度选择 → 连续两次选B型，用户倾向"我们漏掉了什么"的叙事方式
2026-06-25 文案修改 → 矛盾句塞得太密，3组以上读者会疲劳，减到2组效果更好
```

---

## 维护

- **150 行上限。** 超出 → 最旧条目删除（不是归档，是删除）
- **提升条件：** 同一条规律被 ≥3 次验证 → 提升到 `success-patterns.md` 或 `failure-patterns.md`，从本文件移除
- **STEP 3 读取：** 只读最近 20 条（按日期倒序）
- **创作完成后：** 整理归纳（去重、合并同类、删无效条目）
