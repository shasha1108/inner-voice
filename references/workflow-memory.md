## 记录协议

### 什么值得记录

| 情况 | 记录 | 不记录 |
|------|------|--------|
| 用户否决了一个方向 | ✅ 诊断：为什么这个方向失败了 | ❌ "用户不喜欢X风格"（太模糊） |
| 用户确认了某个选择 | ✅ 成功要素：哪个决策被验证有效 | ❌ "这次写得不错"（无信息量） |
| 创作中发现新规律 | ✅ 具体规律+案例 | ❌ "应该注意X"（未经验证） |
| 数据复盘发现 | ✅ 数据+根因+可执行规则 | ❌ "下次要多关注X"（无操作性） |

### 什么不值得记录

- 已经在 SKILL.md 或 references/ 中明确写过的规则（去那里维护，这里不重复）
- 泛化的经验总结（"金句要密""视觉要强"——没有新的案例或精度提升）
- 纯猜测（没有实际数据或用户反馈支撑）
- 单次巧合（只发生一次、没有模式信号的事件）

### 写入格式

```

**事实**：[具体发生了什么]
**根因**：[为什么]

```

## 工作流协作铁律

这些是跨步骤的工作流级别规则，不同于领域知识（领域知识在 account-dna / failure-patterns 中）：

1. **隐喻先确认再进叙事角度**（A-1 → A-2 两段门，不可跳过）
2. **视觉和文案用同一个隐喻**（STEP 4 从 confirmed_metaphor 出发，禁止图文不同隐喻）
3. **配色以停滑力为第一优先级**（先问"信息流里会被看见吗"，再问"和情绪匹配吗"）
4. **文案逐句朗读检验**（写完通读，不拗口、相邻无同音、句间有内在逻辑）
5. **全文只一个隐喻域，禁止域间漂移**（选定隐喻后列出词汇边界，全文不越界）
6. **多版失败后先质疑情绪-媒介适配性**（而非继续换隐喻）
7. **用户给的素材优先级 > skill 默认流程**（用户有明确公式/结构/CTA 时，不用默认流程）

## 数据验证（STEP 9 复盘结论）

### [06-11] 安静的倦怠 — 搜索量 0 词条导致两篇均 < 20 浏览

**事实**：「安静的倦怠」小红书搜索量 0；第一篇浏览 18，第二篇改标题重发仍十几个。视觉低饱和灰绿+卡通小女孩+封面有字 vs 爆款深蓝高对比+剪影+无字。

**根因**：(1) 词条命名为创作者自造词，未搜索验证 (2) 视觉强度与赛道要求倒挂 (3) 复用 2025 爆款公式但未复用其真正驱动力（视觉强度）(4) 账号流量低谷期未识别

**规则**：
- 词条入库前强制小红书搜索验证，搜索量 0 禁止入库
- 连续 2 篇浏览 < 50 → 停止方向 30 天 + 启动破冰模式
- 2025 爆款基因需按季度校准（载体过时改载体，不改基因）

### [06-11] 账号 18 篇塌方群根因

**事实**：18 篇中 11 篇使用抽象/反问/诊断式标题（"过度的自我审视""接受有些热闹""允许自己..."）。视觉待二次验证。

**根因**：(1) 创作者命名而非用户原话 (2) 抽象情绪词而非具象场景词 (3) 封面文字金句（2024 风格在 被降权）(4) 复制旧爆款结构而非理解旧爆款驱动力

**禁用清单**：标题禁用「...的具象化」「放弃X，Y都会为你而来」「过度的X」「允许自己...的人」；视觉禁用低饱和灰绿/米色调、卡通具象人物、封面文字金句。

## 活跃诊断

> 最近 3 条失败诊断。格式同上。

### [06-20] 虚耗型疲惫 × 影子/思绪隐喻 — 根因级否决，视觉辨识度低

**事实**：主题「虚耗型疲惫」，隐喻从「影子在做她没做的事」→「思绪还在加班」，经历多轮视觉方案（影子墙面投影→文字布料→漂浮便签→对比构图→尺度错位玻璃板），用户评价"生成的图片辨识度都很低，没有吸引力，文案也差强人意"。全方向放弃。

**根因**：
1. 「思绪/影子」隐喻缺乏物理形态——不同于漩涡/玻璃盒/废墟（有明确物理形状），"思绪"是抽象状态词，任何视觉化方案都在把抽象概念硬翻译成图像，辨识度天然偏低
2. 影子墙面投影在 MJ 中属于已验证高难度（visual-guide.md 🚫 红线），替代视觉方案（文字布料/漂浮碎片）属于"氛围图"而非"概念钩子"
3. 「思绪加班」文案隐喻本身不够独特——"大脑停不下来"是过于普遍的描述，没有制造出"对，这就是它"的命名感；与账号已有爆款隐喻（漩涡/玻璃盒）相比差距明显

**规则**：
- **「情绪/思绪/感受」等抽象状态词不得直接作为隐喻**：A-1 候选必须先找到有明确物理形状的载体（可以用五字说清的具体物体），否则直接排除
- 动态过程型隐喻（移动中的影子、流动的思绪）→ 图文封面辨识度先天弱，视频模式也无法弥补初始帧的静止辨识度问题
- 「虚耗型疲惫」方向本轮失败，暂停 30 天，重启时需先找到有物理形状的强视觉载体再进 A-1

### [06-15] 跳过 STEP 6-7 直接输出文案 — 流程执行缺陷

**事实**：创作"机器人伴侣"文案时，写完 STEP 3 后直接以确认点 B 格式输出给用户，跳过了 STEP 6-7 评分。用户指出后才补评分。第一版标题 7.6/10 不通过，正文 8.4/10 不通过。

**根因**：把"写完→给用户看"当成了默认动作。正确的默认动作是"写完→评分→不通过就改→通过才给用户看"。SKILL.md 标注了"🚨 内部强制执行 · 不可跳过"但执行时被忽略。

**规则**：
- **STEP 6-7 评分是输出给用户的硬性前置门。** 任何文案在展示给用户之前，必须先完成结构信号检查 + 逐维度评分 + 脚本计算总分 + 阈值判断
- 不通过 → 静默修改 → 重评 → loop_count < 2 时不给用户看中间版本
- loop_count ≥ 2 → 强制通过，附评分报告和未通过原因，告知用户
- **此规则适用于所有内容输出场景**（包括 /推荐 后的选题展开、/配文 模式）

### [06-21] 高敏感 × 情绪传染方向 — 9个候选全落败，机制型题材的隐喻挖掘困境

**事实**：以「情绪传染」为核心感受（用户原话："别人的情绪会传染给我，我可能本来心情很好，结果突然就被别人的负能量影响了"）尝试挖掘高敏感题材隐喻。历经3批共9个候选（散场后还坐着/余震/残灯/雨来之前/原图/没有隔离层/墨水入水/弦的共鸣/温度传导），全部否决。用户评价："强行靠概率拼凑的文字，实际上是没有什么逻辑可言的，思考一下，就觉得很奇怪，不是人类想象出来的东西"。

**根因**：
1. 高敏感描述的是「神经处理机制不同」，不是一种有天然物理形状的情绪状态——漩涡/玻璃盒能命名内耗/孤独，因为那些情绪本身就是那个形状；「高敏感」没有对应天然形状，强行找隐喻必产生外部类比
2. 创作者不是高敏感亲历者，流程没有受众语言前置步骤——导致从概念往下推导而非从真实感受往上命名
3. 问4（情绪真实性）是有效过滤器，但在没有受众原语言作为材料时，过滤后无法产生替代方案，陷入死循环

**规则**：
- **机制型题材（高敏感/内向/ADHD/高共情）不走单一隐喻路径**：改用「场景清单认领」结构——3-7个超具体「只有你懂的瞬间」，无需统一隐喻
- **创作者非亲历者时，STEP 3前必须先做受众语言调研**：找3-5条目标受众自己写的第一人称描述，从原语言提取，不从概念推导
- **问4需配合受众原语言才完整**：问4排除错的，原语言提供对的；没有原语言，问4只能形成否定循环

### [06-21] 高敏感 × 星座图隐喻 — 外部类比强加导致隐喻逻辑崩溃 + 诗意化语言破坏散文逻辑

**事实**：以「高敏感人看整张星座图，普通人只看散星」为核心隐喻创作高敏感题材内容，通过 A-1 视觉可行性三问（有物理形态/美感/本来就存在），但用户指出：(1) 是否看整张星座图与高敏感无内在逻辑关系，是外部类比强加；(2) 结尾段落「这只是你多看了一层的价格。那层里有更多的美，也有更多的重。但那层，只有你看见。」主谓宾混乱，指代不清，转折词逻辑错误，整体割裂不连贯。

**根因**：
1. A-1 三问只验证视觉形式，未验证隐喻的情绪真实性（内认度）——外部类比与内部命名都能通过三问
2. 「星座图」类比需要读者接受「普通人只看散星」的外部前提，而高敏感者的内部体验不是「看见更多连线」，而是「感受更强烈」——类比在情绪逻辑上不成立
3. 用诗意化语言强行拉高「价格/那层/美和重」的文学感，但绕过了基本的中文散文逻辑连贯性（指代/转折词/名词化抽象词的指向）

**规则**：
- **A-1 通过三问后，必须过问4（情绪真实性内认测试）**：目标读者第一次读到隐喻，反应是「就是这样」还是「好吧如果这么理解」？后者=外部类比→直接排除
- **内部命名 vs 外部类比**：漩涡/玻璃盒/废墟是内部命名（情绪本来就长这样）；星座图是外部类比（需要接受外部假设才能感受）→ 候选隐喻必须来自对目标情绪的身体经验提取，而非从视觉美感出发
- **散文逻辑自检（STEP 5 新增）**：诗意化段落逐句检查：指代是否明确/转折词是否符合逻辑/名词化抽象词是否有指向
